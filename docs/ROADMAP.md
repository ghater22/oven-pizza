# ROADMAP

## الإصدار الحالي (v1.0 — نطاق هذا المشروع)

- إدارة إيرادات ومصروفات لفرعين.
- Dashboard + تحليلات منتجات/أوقات + رؤى ذكية قائمة على قواعد.
- تقارير + تصدير PDF/Excel.
- مستخدم واحد (المالك).

## بعد v1.0 (مقترحات مستقبلية — تحتاج موافقة صريحة قبل التنفيذ)

- **أدوار إضافية**: مستخدم محاسب/مدخل بيانات بصلاحيات محدودة (يتطلب توسيع قواعد الأمان وربما Cloud Functions لإدارة الصلاحيات → قرار مالي يحتاج موافقة لأنه قد يستلزم خطة Blaze).
- **إضافة فروع جديدة**: البنية تدعم ذلك بدون تعديل مخطط البيانات (branch جديد = وثيقة جديدة في `branches`).
- **صور المنتجات**: تفعيل Firebase Storage (ضمن الحصة المجانية أيضًا لحجم صغير).
- **تنبيهات فورية (Push Notifications)**: عبر Expo Notifications (مجاني) عند حدوث انحراف كبير (مثلاً مصروف غير معتاد) دون انتظار فتح التطبيق.
- **نسخ احتياطي/تصدير كامل للبيانات**: تصدير كل قاعدة البيانات إلى Excel لأرشفة سنوية.
- **مقارنة تنبؤية بسيطة**: توقع نهاية الشهر بناءً على المعدل اليومي الحالي (حسابي بسيط، ليس ML).
- **نشر على المتاجر**: فقط إذا رغب المالك بذلك مستقبلًا (رسوم Google Play/Apple تتطلب موافقة صريحة مسبقة).

## غير مخطط له إطلاقًا ضمن فلسفة هذا التطبيق

- نظام طلبات للعملاء.
- تطبيق عملاء عام.
- نموذج تجاري SaaS لعدة مطاعم.

# SPRINT PLAN

تنفيذ متتابع بدون توقف بين السبرنتات لموافقات روتينية (بحسب تعليمات المالك)، مع **commit محلي واحد على الأقل بعد كل سبرنت** (بدون `git push`).

## Sprint 0 — الأساس
- ملفات التوثيق التسعة (هذا المجلد).
- مشروع Expo + TypeScript + expo-router + NativeWind.
- Firebase wiring (config من `.env`) + `firestore.rules` + `firestore.indexes.json`.
- الثيم (Light/Dark) + خط Cairo + فرض RTL.
- الشعار والأيقونات (SVG → PNG).
- هيكل تنقل أساسي + شاشة تسجيل دخول (UI فقط أولًا ثم مربوطة بـ Firebase Auth).

**معيار القبول**: `npx expo start` يعمل بدون أخطاء، شاشة الدخول تظهر بـ RTL بالثيم الصحيح.

## Sprint 1 — Auth + Dashboard + الفروع
- تسجيل دخول فعلي (Firebase Auth) + حارس مصادقة (redirect لو غير مسجّل).
- شاشة إدارة الفروع (عرض/تعديل اسم، لا حذف لتفادي فقد بيانات تاريخية).
- Dashboard: دخل/مصروف/ربح اليوم لكل فرع وللإجمالي، مقارنة الفرعين، تحديد "أفضل فرع".

**معيار القبول**: تسجيل دخول حقيقي يعمل، Dashboard يعرض 0 بشكل صحيح عند عدم وجود بيانات (Empty State) لا كسر بالتطبيق.

## Sprint 2 — الإيرادات
- نموذج إضافة إيراد (فرع، منتج، كمية، سعر وحدة محسوب تلقائيًا من سعر المنتج قابل للتعديل، وقت).
- قائمة إيرادات قابلة للفلترة بالتاريخ والفرع، تعديل وحذف.

## Sprint 3 — المصروفات
- نموذج إضافة مصروف (فرع، تصنيف من قائمة ثابتة، مبلغ، وقت، ملاحظة).
- قائمة مصروفات قابلة للفلترة، تعديل وحذف.

## Sprint 4 — تحليل المنتجات
- شاشة تعرض: الأكثر/الأقل مبيعًا، الأعلى ربحًا، مقارنة فرع مقابل فرع لنفس المنتج، فلترة بالفترة.

## Sprint 5 — تحليل الأوقات
- رسم بياني بالساعة (24 عمود) ورسم بياني بيوم الأسبوع (7 أعمدة)، تمييز الذروة/الركود بصريًا.

## Sprint 6 — الرؤى الذكية
- تفعيل محرك القواعد (`ANALYTICS_MODEL.md` §6) وعرض أعلى الرؤى في `InsightBanner` على الـ Dashboard.

## Sprint 7 — التقارير والتصدير
- شاشة تقارير: اختيار نوع (يومي/أسبوعي/شهري/فرع/منتج/مصروف) وفترة.
- زر تصدير PDF (قالب HTML عربي RTL عبر `expo-print`) وزر تصدير Excel (`xlsx`) + مشاركة الملف (`expo-sharing`).

## Sprint 8 — تلميع نهائي
- مراجعة كل الشاشات: حالات فارغة، حالات تحميل، تباين الألوان في Dark/Light.
- فحص TypeScript كامل (`tsc --noEmit`) بدون أخطاء.
- تحديث `TASKS.md` وأي توثيق تغيّر أثناء التنفيذ.

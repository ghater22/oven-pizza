# UI DESIGN — التصميم

## اللغة والاتجاه

- عربية بالكامل، RTL إجباري (`I18nManager.forceRTL(true)` عند الإقلاع).
- خط: **Cairo** (Google Fonts، مجاني) — عناوين Bold/SemiBold، نص عادي Regular.

## الهوية البصرية

**الشعار**: قرص بيتزا مبسّط داخل فرن طوب (قوس فرن + شعلة صغيرة)، أسلوب Flat/Vector، بلا تفاصيل زائدة — مصمم كـ SVG قابل للتوليد إلى أيقونات Expo (icon/adaptive-icon/splash).

## المكونات الأساسية

- **StatCard**: رقم كبير + عنوان + أيقونة + مؤشر اتجاه (▲/▼ % مقارنة بالفترة السابقة).
- **BranchSwitcher**: تبديل سريع (الكل / فرع 1 / فرع 2) أعلى كل شاشة بيانات.
- **ChartCard**: بطاقة تحتوي رسم بياني (Bar/Line/Donut) + عنوان + فترة زمنية قابلة للتغيير.
- **InsightBanner**: شريط ملون (تحذير/إيجابي) يعرض جملة رؤية ذكية واحدة.
- **EmptyState**: رسمة + نص عندما لا توجد بيانات بعد (بدل شاشة فارغة مربكة).
- **PrimaryButton / IconButton / FAB**: FAB دائري ثابت أسفل يمين (RTL: يسار) لإضافة إيراد/مصروف بسرعة.

## هيكل التنقل (Navigation)

Bottom Tabs (RTL-aware، الأيقونات تُقرأ من اليمين لليسار):

1. **الرئيسية** (Dashboard)
2. **الإيرادات**
3. **المصروفات**
4. **التحليلات** (منتجات/أوقات)
5. **التقارير**

زر الإعدادات (تبديل Dark/Light، تسجيل الخروج، إدارة الفروع/المنتجات) في أعلى الشاشة الرئيسية كأيقونة، وليس تبويبًا مستقلًا لتوفير مساحة.

## مبادئ تجربة الاستخدام

- إدخال إيراد/مصروف عبر نموذج بسيط بخطوة واحدة (Bottom Sheet)، لا تنقل بين صفحات متعددة.
- الأرقام دائمًا بتنسيق عملة واضح (فاصلة آلاف) ولا تقريب مضلل للأرباح.
- كل رسم بياني يحمل عنوانًا يوضح الفترة (اليوم/الأسبوع/الشهر) بشكل بارز.
- حالات التحميل: Skeleton بدل دوّار تحميل فارغ.
- إمكانية الوصول: كل عنصر تفاعلي حجمه ≥ 44×44، تباين ألوان كافٍ، دعم قارئ الشاشة عبر `accessibilityLabel` بالعربية.
